\begin{frame}
\frametitle{Le dénis de service (DOS) qu'est ce que c'est?}
\begin{itemize}
\item Une attaque par déni de service consiste ŕ empęcher un systčme informatique de fonctionner correctement.
\bigskip 
\item On \alert{inonde un serveur de requętes}, qui ne peut plus répondre ŕ de nouvelle requęte légitime.
\bigskip 
\item L'inondation  peut monopoliser d'autre type de ressource: nombre de connexions simultanées ou la mémoire disponible.
%\item Les dénis de service les plus intéressants sont ceux qui peuvent ętre effectués de fa\c con anonyme.
\end{itemize}
\end{frame}

\begin{frame}[t]
\frametitle{SYN flooding - Rappel du mécanisme d'ouverture de connexion TCP}


\begin{columns}[t]
 \begin{column}{0.3\textwidth}
\begin{center}
\includegraphics[width=3.5cm]{tcp-initialization}
\end{center}
\end{column}
 \begin{column}{0.7\textwidth}
%L'initialisation d'une connexion se fait comme suit:
\begin{small}
\begin{itemize}
\item Le  client envoie un paquet SYN (le bit SYN = 1  le bit ACK = 0). Le numéro de séquence $x$ est le numéro de séquence initial (ISN).
Le numéro d'acquittement est zéro  car l'ISN du serveur est inconnu.
%\item Le numéro d'acquittement est zéro car le client ne connaît pas encore les numéros de séquence des données provenant du serveur.
\item Le serveur répond par un paquet SYN-ACK (le bit SYN = 1 et le bit ACK = 1). Le numéro de séquence $y$ est le  numéro de séquence initial (ISN) choisit par le serveur, le numéro d'aquitement est $x+ 1$.
\item Le client répond au SYN-ACK par un segment ACK (SYN est ŕ 0 et le bit ACK est ŕ 1), le numéro de séquence est  $x+1$, le numéro d'ACK est $y + 1$.
\end{itemize}
\end{small}
\end{column}
\end{columns}
\end{frame}

\begin{frame}
\frametitle{SYN flooding - description}
 Le SYN flooding est l'une des plus anciennes attaques de déni de service:
\begin{itemize}
\item {\color{blue} Principe :} L'attaque consomme toutes les ressources de la couche TCP d'un ordinateur.
\medskip
%Comme nous l'avons vu ŕ la section 3.1 les paquets SYN sont utilisés pour initier les connexions TCP
\item {\color{blue} But :} Lors de l'initialisation d'une connextion TCP, ŕ la réception du paquet SYN,  \alert{le serveur met cette connexion dans une file d'attente}.
\medskip
\item Cela  maintient la file d'attente constamment remplie, le serveur refuse alors systématiquement de nouvelle connexion.
\medskip

\item L'attaquant peut utiliser des adresses IP arbitraires vu qu'il n'est pas intéressé par réellement ouvrir connexion TCP.
\end{itemize}
\end{frame}



\begin{frame}
\frametitle{SYN flooding - parades}
\begin{itemize}
\item {\color{blue} Stratégie 1 :} Les connexions en attente sont retirées de la file d'attente si l'ACK attendu n'arrive pas dans un délai raisonnable.
\bigskip

\item {\color{blue} Stratégie 2 :} On  en utilisant des SYN Cookies (Linux, FreeBSD) : 
\smallskip

\begin{itemize}
\item Le serveur ne maintient plus de file d'attente,
\item Les informations concernants la connexion sont encodées dans l'ISN, avec un mini-mécanisme de signature pour éviter des falsification.
\item A la réception de l'ACK le serveur récupčre les informations des ``vrai'' clients et peut poursuivre la communication TCP en mode connecté.
\end{itemize}
\end{itemize}
\end{frame}

\begin{itemize}
\item L'attaque smurf consiste ŕ noyer la victime dans un flux de de réponses ICMP.
\item {\color{blue} Rappel :} ICMP est un protocole qui sert entre autre ŕ vérifier qu'une machine est atteignable sur le réseau.
\item Quand une machine re\c coit un \textit{\color{red} ICMP echo request} elle envoie un paquet \textit{\color{red}ICMP echo reply} ŕ l'adresse IP source indiquée dans le paquet
 \begin{center}

\end{itemize}
\end{frame}


\begin{frame}
\frametitle{Attaque smurf}
\begin{itemize}
\item Pour attaquer une cible avec des {\color{red} réponses ICMP}, le pirate inscrit l'adresse cible comme adresse IP source dans l'en-tęte IP du paquet ICMP de la requęte.
\item La destination de la requęte enverra ainsi sa réponse ŕ la cible.
\item La cible ne saura d'ailleurs pas qui lui veut du mal.
\item Pour amplifier son attaque le pirate peut envoyer sa requęte ŕ une adresse de diffusion réseau.
\end{itemize}

\subsection{IP spoofing}

\begin{frame}
\frametitle{IP spoofing}

Le spoofing d'IP consiste en la \alert{falsification d'adresse IP.}

%Dans de nombreux cas des paquets de données sont traités différemment en fonction de leur adresse IP d'origine
\begin{itemize}
\item Un routeur ou un pare-feu peut laisser des paquets qui proviennent de sites \textit{amis}. 
\medskip

\item Certaines applications offrent des privilčges aux connexions qui émanent de sous-réseaux connus.
\medskip
\item Par exemple \texttt{rsh} d'Unix peut permettre l'exécution de commande sans authentification préalable (s'assure que l'IP est dans la liste d'adresses IP amies). 

\end{itemize}

%Un pirate peut usurper cette confiance en falsifiant l'adresse source de ces paquets.
\end{frame}


\begin{frame}
\frametitle{Le Spoofing d'IP lors d'une session TCP}

Il est particuličrement difficile de réussir une attaque par spoofing lors d'une session TCP:
\begin{itemize}
\item Si le pirate remplace son IP par une autre adresse, la cible enverra ses réponses ŕ l'adresse falsifiée.
\item C'est gęnant car l'initialisation d'une session TCP passe par une séquence de SYN,SYN-ACK,ACK.
\item Si le pirate ne peut pas répondre au SYN-ACK, il ne peut envoyer des données.
\end{itemize}
Dans certains cas on peut contourner ce problčme.
\end{frame}

\begin{frame}
\frametitle{TCP spoofing dans le męme sous-réseau}

Si l'attaquant est dans le męme sous-réseau que l'adresse falsifiée:
\begin{itemize}
\item il pourra observer les réponses męme si elles ne lui sont pas destinées.
\item il verra l'ISN que le serveur propose dans le SYN-ACK.
\item il pourra faire une réponse ACK valide, et envoyer des données avec une fausse adresse IP.
\end{itemize}
\end{frame}


\begin{frame}
\frametitle{TCP spoofing ŕ l'aveugle}
\begin{itemize}
\item Si l'attaquant n'est ni dans le sous-réseau de la cible ni dans celui de la machine dont l'adresse IP est usurpée il ne n'observera pas le paquet SYN-ACK.
\bigskip

\item Pour qu'il puisse répondre sans recevoir le SYN-ACK, il faut qu'il devine l'ISN contenu dans le SYN-ACK:
\begin{itemize}
\item Si la méthode de génération de l'ISN est simple,  quelques échantillons peuvent suffire pour générer les prochains ISN.
\item De vieilles versions  de Windows et d'Unix utilisaient de simple incrément.
%\item Aujourd'hui des générateurs aléatoires plus efficaces ont été mis en place. L'attaquant peut toujours tenter sa chance en envoyant plusieurs paquets avec différents ISN, les mauvais numéros seront ignorés le bon accepté.
\end{itemize} 
\end{itemize} 
\end{frame}

\subsection{Sniffing}

\begin{frame}
\frametitle{Sniffing}
\begin{itemize}
\item Le sniffing consiste ŕ capturer du trafic qui circule sur une partie du réseau.
\item Le but du sniffing est d'exploiter le fait que beaucoup de méthode d'authentification transmettent les mots de passe en clair sur le réseau.
\item C'est utilisé par les pirates pour étendre leurs attaques/réseaux de bots en prenant le contrôle de nouvelles machines.
\begin{itemize}
\item Dans les réseaux locaux sans fils, c'est plus facile car le pirate a accčs ŕ toutes les données qui transitent.
\item Dans réseaux locaux câblés en général les machines sont connectés directement ŕ des hubs et \alert{donc les trafics sont cloisonnés}.
\end{itemize}
\end{itemize}
\end{frame}

\subsection{Vol de session}
\begin{frame}

\begin{block}{Vol de session : }
%Le \textit{\color{red} session hijacking} ou \textit{\color{red}vol de session} est un moyen de pénétrer dans un systčme sans avoir besoin de connaître login et password:

\item Un vol de session TCP  peut provoquer des \alert{avalanches de paquet d'acquitement} : cela perturbe l'ordre normal des numéros de séquence du client.
\end{itemize}

\item En effet chaque requęte pour une page web (page HTML, image, etc. ) est traitée de maničre indépendante.
\item Les développeur web contourne ce problčme en identifiant les utilisateurs grâce ŕ des cookies et des données stockées sur une base de données.

\begin{itemize}

\begin{frame}
\frametitle{Qu'est ce qu'un pare-feu}
\begin{itemize}
\item Les pare-feux sont des composants d'un réseau utilisés chaque fois que l'on désire interconnecter deux réseaux possédant un niveau de sécurité différent.

\item La fonction primitive d'un pare-feu est d'éviter la propagation d'un incendie.

\item De fa\c con analogue dans le contexte des réseaux, un pare-feu doit prévenir la propagation d'une attaque entre les différents réseaux qu'il relie.

\end{itemize}
\end{frame}

\begin{itemize}
\item Un systčme de sécurité n'est efficace que si tous les utilisateurs adhčrent ŕ ses principes.

\subsection{Filtrage}



\begin{frame}
\frametitle{Filtrage : généralité}
\begin{itemize}
\item Le filtrage consiste ŕ autoriser ou bloquer du trafic ŕ entrer ou sortir non d'une partie du réseau.
\item  Le filtrage consiste ŕ applique certaines rčgles concernants les critčres suivants:
\begin{itemize}
\item Adresse IP source ou destination.
\item Protocole (TCP, UDP, ICMP, ...) et port.
\item Drapeaux et options : TCP SYN, ACK, ICMP.
\end{itemize}

\item {\color{blue} Exemple :} le filtage d'adresse IP permet de contrer des attaques par spoofing.

\item {\color{blue} Exemple :} le filtage sur les drapeaux SYN et ACK permet de contrer une attaque SYN.


\item {\color{blue} Exemple :} le filtage sur ICMP peut autoriser des \texttt{echo-requests} que dans une direction (sortante).
\end{itemize}

\end{frame}





\begin{frame}
\frametitle{ Filtrage : généralité}

\begin{itemize}
\item Un pare-feu est configuré via une liste de rčgles de filtrage.

\item Pour chaque rčgle on doit spécifier les caractéristiques du paquet (adresses source et/ou destination, ports, drapeaux, \ldots) auxquelles la rčgle s'applique.

\item Chaque rčgle contient aussi l'action ŕ effectuée avec les paquets sélectionnés par exemple les laisser passer ou les détruire. 


\item Pour chaque paquet re\c cu le pare-feu parcourt la liste des rčgles jusqu'ŕ ce qu'il en trouve une qui s'applique. 

\item L'ordre dans lequel ces rčgles sont lues est donc \alert{trčs important!}.
\end{itemize} 

\end{frame}

\begin{frame}
\frametitle{Courrier électronique indésirable (SPAM)}

La notion de courrier \textit{indésirables} étant subjective  la classification d'un courrier envoyé en masse comme du spam est délicate:
\begin{itemize}
\item La CNIL propose la définition suivante:

\begin{itshape}
Constituent des spam les messages adressés sur la base d'une collecte irréguličre de courriel soit au moyen de moteurs de recherche dans les espaces publics de l'Internet soit que les adresses aient été cédées sans que les personnes en aient été informées et sans qu'elles aient été en mesure de s'y opposer ou d'y consentir. 
\end{itshape}
\item L'aspect concernant la collecte d'adresse est discutable, le Syndicat National de la Communication Directe propose la définition suivante

\begin{itshape}
Adresses collectées de maničre déloyale, envoi ne respectant pas les modalités d'utilisation précisées lors de la collecte; courriel ne permettant pas une désinscription sélective ou totale. 
\end{itshape}
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Historique}

Il est difficile de dater l'apparition du premier spam:
\begin{itemize}
\item Certains l'attribue ŕ Gary Thuerk employé chez DEC qui envoya en 1978 un courrier publicitaire ŕ l'ensemble des utilisateurs d'ARPAnet ($\cong$ une centaines de personnes).


\item D'autres l'attribue ŕ L. Canter et M. Siegel, deux avocats spécialisé dans les affaires d'immigration, qui ont envoyé en 1994 un seul spam ŕ plusieurs millions de personnes. 
\item Ils ont envoyés sur 6000 groupes de diffusion un message faisant la publicité de leur service pour l'obtention de la \textit{green card}, ils ont recu 25 000 réponses, leur rapportant 100 000 dollars.
\item En contrepartie ils ont été plusieurs fois condamnés et męme Canter a été radié du barreau en 1997. 
\end{itemize}

\end{frame}



\begin{frame}
\frametitle{Technique de diffusion du spam - Relais ouverts}

\begin{itemize}
\item Le fait de recevoir un spam est surtout une nuisance.
\item C'est la fa\c con dont il est distribué qui crée des problčmes de sécurité.
\item L'envoi d'email ŕ des millions d'adresses nécessite des ressources importantes.
\item Un serveur relais est un serveur de courriel d'une entreprise qui est utilisé par des utilisateurs extérieurs (auteurs de spam) pour envoyer leur propre courriel.
\end{itemize}
\end{frame}





\begin{frame}
\frametitle{Diffusion du spam : Serveur Relais}
\begin{itemize}
\item Il est plus efficace pour l'auteur d'un spam qu'il l'envoie une copie ŕ plusieurs serveurs de courrier (SMTP) et une liste de plusieurs milliers d'adresse email.
\item Pour l'entreprise dont le serveur mail est abusé:
\begin{itemize}
\item Durant le temps de l'envoie des mails (qq minutes ou heures) le serveur est surchargé par l'envoie des spams. 
\item Le serveur ne peut męme plus gérer les emails interne ŕ l'entreprise.
\item Si l'entreprise est de petite taille, la connexion internet est elle aussi surchargée. 
\item Le disque dur du serveur se remplit de log de confirmation de courriers bien délivrés.
\item La boîte de courriel de l'admin se remplit de message d'erreur pour les adresses email invalides. 
\item Le FAI de l'entreprise peut menacer de couper la connexion internet s'il recoit des plaintes de destinataires de spam. 
\item Le serveur abusé risque d'ętre placé sur liste noire, et les futurs courriels  ``normaux'' risqueront d'ętre rejetés.
\end{itemize}
\end{itemize}

\begin{frame}
\frametitle{Diffusion du spam - Rčgles pour éviter l'utilisation d'un serveur relais}

%Un serveur relais est un serveur de courriel d'une entreprise qui est utilisé par des utilisateurs extérieurs (auteurs de spam) pour envoyer leur propre courriel.


Pour éviter qu'un serveur de courriel ne devienne un relais ouvert on doit appliquer les rčgles suivantes:
\begin{enumerate}
\item Pour qu'un message soit accepté par le serveur de courrier, soit l'adresse de l'expéditeur, soit celle du destinataire doit appartenir au domaine du serveur : cela évite l'utilisation du serveur comme  relais.
\item Seuls les systčmes appartenant au męme domaine que le serveur peuvent envoyer du courrier avec un expéditeur faisant partie du domaine : cela empęche des utilisateurs extérieurs de se faire passer pour des utilisateurs locaux.
\end{enumerate}

\end{frame}



\begin{frame}
\frametitle{Diffusion du spam :Spambots}

Aujourd'hui une part importante du trafic de spam provient de spambot (ou robot spammeur):
\begin{itemize}
\item Un \textit{robot} est une machine compromise qui obéit aux ordres d'un maître.
\item La compromission d'une machine peut se faire grâce ŕ un virus ou via des failles de sécurité de la machine.
\item Un robot élémentaire consiste en un relais SMTP ouvert ŕ tous et en particulier aux auteurs du spam
\item Un robot plus élaboré se connecte ŕ un serveur de discussion (par exemple un canal IRC = Internet Relay Chat) et se met en écoute des commandes du maître.
Le maître se connecte sur le canal et envoie ses ordres ŕ tous les robots connectés au canal.
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Diffusion du spam : Messagerie gratuite en ligne}

Une derničre technique pour diffuser du spam consiste ŕ abuser des messageries gratuites en lignes (Hotmail, Yahoo mail, gmail, ...):
\begin{itemize}
\item Tous ces webmails limitent le nombre de courriels qui peuvent ętre envoyés.
\item Les diffuseurs de spam créent des scripts pour ouvrir automatiquement des milliers de nouveaux comptes.
\item Les scripts alors envoient pour chacun des comptes une quantité limité de courriels.
\item Les opérateurs de messagerie en ligne se protčge contre les créations de comptes automatiques par des \textit{captchas}, devinettes graphiques sensées ętre déchiffrable que par un humain.  
\end{itemize}
\end{frame}

\subsection{Lutte contre le spam}

\begin{frame}
\frametitle{Filtrage}

Le filtrage peut porter 
\begin{itemize}
\item sur le contenu contient-il les mots clefs Viagra, XXL, etc. ;
\item des paramčtres de l'en-tęte du courrier (temps de transit, ...) ;
\item le format des messages (titre du message en lettre capitales, ...).
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Filtrage : Spamassassin}


Spamassassin est un logiciel de lutte contre le spam qui utilise des rčgles de filtrage:
\begin{itemize}
\item Des centaines de rčgles peuvent ętre appliquées lors du traitement ;
\item A chaque rčgle est attribuée un poids relié l'importance de la rčgle.
\item A l'issu de l'analyse les poids sont combiné pour donner un score au message : au dessous d'un certain seuil les message est accepté, au dessous le message est rejeté.
\end{itemize}

Ce type d'approche a clairement des limites: si les rčgles sont mal ajustés des spams peuvent passer ŕ travers le filtre et des mails réguliers peuvent ętres rejetés par erreur.
\end{frame}

\begin{frame}
\frametitle{Liste noires et liste blanches}

Une approche de lutte contre le spam consiste ŕ utiliser des listes noires et blanches:
\begin{itemize}
\item Une liste noire contient des adresses IP de machines qui envoient du spam.% Il peut s'agir de machines utilisées exclusivement pour cela ou des machines compromises qui le font ŕ leur insu.
Exemple de liste : celle gérée par Spamhous Project.
\item  Une liste blanche contient des adresses IP, noms de domaines ou expéditeurs de confiance.% c'est ŕ dire qui sont connus pour ne pas envoyer de spam. Une liste blanche est généralement.
 Utilisée en complément d'une autre méthode de lutte contre le spam pour éviter le rejet de mail de correspondant régulier.

\end{itemize}

\end{frame}

\begin{frame}
\frametitle{Signature de spam}
\begin{itemize}
\item Un spam est envoyé ŕ des millions d'exemplaire sur des plage de temps relativement longue (plusieurs jours ŕ plusieurs mois) 
\item On peut mutualiser les efforts : des bases de données de signature de spams (signature = portion  discriminante du message)  sont stockées sur des serveurs.
\item  Il est alors possible d' interroger la base de données pour vérifier si un courriel suspect est un spam ou non.
\item L'alimentation de la base de données se fait soit de facon collaborative ou par utilisation/diffusion d'adresse mail cible facile de spammeurs.
\end{itemize}
\end{frame}



\begin{frame}
\frametitle{Liste grise}

Cette technique consiste ŕ détecter les comportement ``bourrin'' de certains diffuseurs de spams:
\begin{itemize}
\item Lors de la réception d'un email d'un expéditeur: le mail est bloqué, l'adresse IP et l'adresse email de l'émetteur est stockée et l'email est jeté.
\item Le protocole SMTP prévoit, lors de la perte d'un message, la ré-émission du message par le serveur SMTP expéditeur.
\item Lorsque le message est recu pour la seconde fois, le message est alors remis dans la boîte au lettre du destinataire.
\item Les serveurs qui envoient des spams par grosse quantité ne se soucis pas de ré-émettre des messages perdus ou rejetés. Ce sont eux que détecte la technique de la liste grise.
\item Cette technique est inefficace contre les spambots.
\end{itemize}
\end{frame}

\subsection{Modčle économique du spam}

\begin{frame}
\frametitle{Modčle économique du spam}

Le modčle économique de Canter et Siegel dans les années 90 était simple:
\begin{itemize}
\item l'envoie de spam avait pour objectif de faire connaître leur activité, 
\item les bénéfices qu'ils en retiraient provenaient ensuite exclusivement de leur travail d'avocat.
\end{itemize}
\end{frame}

\begin{frame}
\frametitle{Modčle économique du spam}
Aujourd'hui la situation est plus complexe, une économie s'est mise en place:
\begin{itemize}
\item Ceux qui collectent les adresses peuvent vendre ou louer leur liste d'adresse. Cette activité
est légale car les adresses sont collectées sur le domaine publique et lors de la vente 
les acheteurs attestent du fait qu'ils ne vont pas utiliser les adresses pour envoyer des pourriels.

\item Ceux qui envoient du spam sont rémunérés en louant leurs serveurs ou leurs services comme la mise ŕ disposition de réseau de spambot.

\item Les auteurs des spams ŕ proprement dit qui diffusent une information pouvant avoir un intéręt directe vente en lignes engendrés par le message ou la publicité.

\item Le but du spam lui męme peut servir ŕ collecter des adresses.

\end{itemize}


\end{frame}
